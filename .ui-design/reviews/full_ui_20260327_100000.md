## Summary

Both codebases are well-structured with consistent dark theming and clear component boundaries. The web app has significant accessibility gaps (keyboard navigation, ARIA roles, focus management). The mobile app has critical safe area issues and several broken navigation paths. Performance optimizations are needed in both search and tree rendering.

## Critical Issues (7)

### WEB-1: Loading screen ignores theme preference
**File:** `packages/client/src/App.tsx:189-194`
Hardcoded `bg-surface-950` — light mode users see a black flash before app mounts.

### WEB-2: Modals missing role="dialog", aria-modal, focus trap
**File:** `packages/client/src/pages/AccountSettingsPage.tsx:104-111`
Also: AdminPage.tsx. Screen readers don't announce dialogs. Focus escapes behind backdrop.

### WEB-3: Sidebar renderNode is a function, not a component — O(n) DOM reconstruction
**File:** `packages/client/src/components/Sidebar/Sidebar.tsx:257-363`
Entire tree re-created on every note selection. Extract to React.memo SidebarNode.

### MOBILE-1: Missing SafeAreaView on NoteScreen, SearchScreen, TagsScreen, GraphScreen
**Files:** `note/[...path].tsx:213`, `search.tsx:133`, `tags.tsx:215`, `graph.tsx:352`
Hardcoded `paddingTop: 36` is wrong on Dynamic Island (59pt) and notch (44pt) devices.

### MOBILE-2: Breadcrumbs navigation broken — all segments go to Notes root
**File:** `packages/mobile/src/components/Breadcrumbs.tsx:29`
Every breadcrumb segment navigates to `/(app)/(tabs)/notes` regardless of folder depth.

### MOBILE-3: Daily and Templates routes missing (app) prefix
**Files:** `daily.tsx:91`, `templates.tsx:86`
`router.push('/note/...')` instead of `/(app)/note/...` — navigation fails.

### MOBILE-4: EditorBridge XSS-vulnerable content injection
**File:** `packages/mobile/src/webview/EditorBridge.tsx:111-120`
`JSON.stringify(content)` injected directly into JavaScript string. Use postMessage instead.
